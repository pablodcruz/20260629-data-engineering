# Data Engineering Cohort

This repository contains the working materials for the Data Engineering cohort that follows the Data Foundations track.

This is a smaller continuation cohort: 11 of the original 22 students are moving forward into the data engineering phase.

## Prerequisite Track

This cohort builds on the Data Foundations course:
[20260523-Data-Foundation](../20260523-Data-Foundation)

## Cohort Focus

By the end of this cohort, students should be able to:

- Design and build repeatable data pipelines.
- Move data between files, APIs, databases, and warehouse-style storage.
- Use orchestration concepts to schedule and monitor work.
- Model analytical data for reporting and downstream use.
- Apply data quality checks, logging, and failure handling.
- Explain pipeline architecture and tradeoffs clearly.

## Student Expectations

- Keep code organized, readable, and committed regularly.
- Treat data engineering work as production-oriented work: validate inputs, log important events, and handle failures intentionally.
- Do not commit secrets, passwords, API keys, `.env` files, virtual environments, or large generated datasets.
- Be ready to explain the architecture of your pipeline, not only the code.
